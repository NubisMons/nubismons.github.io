玩转 Aarch64 架构超算：A100 环境配置与 VS Code 丝滑开发指南

背景：在典型的国产化或定制化超算集群中，登录节点与计算节点往往采用 ARM (aarch64) 架构。配合 NVIDIA A100 40GB 顶级算力，这类环境在驱动、库兼容性及网络限制上有一套独特的逻辑。本文记录了从零开始搭建深度学习环境并实现 VS Code 直连的完整闭环。

1. 终端与 VS Code 远程连接配置

1.1 SSH 基础配置

在本地电脑的 ~/.ssh/config（Windows 为 C:\Users\用户名\.ssh\config）中添加以下配置。通过 ProxyCommand 实现跳板机逻辑，让 VS Code 可以一键直达计算节点。

# === 登录节点 (用于日常文件管理) ===
Host BSCC-Login
  HostName ssh.cn-zhongwei-1.paracloud.com
  Port 22
  User scx7784@BSCC-N32-H
  IdentityFile ~/.ssh/id_rsa

# === 计算节点 (GPU 专用，HostName 需根据每日申请到的节点修改) ===
Host BSCC-Compute
  HostName paraai-n32-h-01-agent-13
  User scx7784
  ProxyCommand ssh scx7784@BSCC-N32-H@ssh.cn-zhongwei-1.paracloud.com -p 22 -W %h:%p
  IdentityFile ~/.ssh/id_rsa

# === 全局防断开配置 ===
Host *
  ServerAliveInterval 20
  HostkeyAlgorithms +ssh-rsa
  PubkeyAcceptedAlgorithms +ssh-rsa

1.2 免密登录避坑

如果连接时反复弹出密码框，需将本地公钥（id_rsa.pub）手动写入服务器。

  1. 在服务器终端执行：nano ~/.ssh/authorized_keys
  2. 粘贴公钥，保存退出。
  3. 关键步骤：执行 chmod 600 ~/.ssh/authorized_keys 锁定权限，否则 SSH 服务出于安全考虑会拒绝免密登录。

2. 深度学习环境搭建 (Aarch64 专项)

由于计算节点严格禁网且架构特殊，传统的 pip install 无法使用或容易装错架构。

2.1 清理环境冲突

若个人目录下存在旧的 Miniconda 导致环境变量混乱，需在 ~/.bashrc 中注释掉旧的 conda initialize 块，改用系统推荐的加载方式：

1
2

# 加载系统官方 Conda
module load miniforge3/24.1

2.2 创建环境与离线安装 PyTorch

在登录节点（有网）进行操作：

  1. 创建环境：
1
2

conda create -n my_gpu_env python=3.9 -y
source activate my_gpu_env
  1. 本地离线安装： 从管理员维护的公共包目录（如 /home/bingxing2/apps/package/pytorch）中寻找对应版本。
1
2
3

# 以 PyTorch 2.0.1 + CUDA 11.8 + Python 3.9 为例
pip install /home/bingxing2/apps/package/pytorch/2.0.1+cu118_cp39/*.whl \
    -i https://pypi.tuna.tsinghua.edu.cn/simple --default-timeout=1000

注意：使用 -i 参数是为了联网下载必要的 Python 依赖包（如 Pillow），同时配合 --default-timeout 应对不稳定的跨境网络。

2.3 修复 NumPy 2.0 兼容性警告

若运行代码报 A module that was compiled using NumPy 1.x cannot be run in NumPy 2.0，需强制降级：

1

pip install "numpy<2" -i https://pypi.tuna.tsinghua.edu.cn/simple

3. 日常开发工作流 (Daily Routine)

超算的计算节点是动态分配的，每天的“门牌号”（Agent ID）都会变化。

Step 1: 申请资源

在登录节点终端提交作业申请（sbatch 或 srun），通过 squeue -u 用户名 获取当天分配到的节点编号（如 agent-25）。

Step 2: 环境挂载 (计算节点)

进入计算节点后，必须手动挂载底层驱动库，否则 import torch 会报 libcudnn.so 缺失错误：

1
2
3
4

ssh paraai-n32-h-01-agent-25   # 跳转到当日节点
module load miniforge3/24.1     # 加载 Conda 工具
module load compilers/cuda/11.8 # 加载 CUDA 驱动库 (关键！)
source activate my_gpu_env      # 激活你的专属环境

    b = torch.randn(10000, 10000, device='cuda')

4. 结语与注意事项

  • 计费提醒：训练结束后务必 scancel [JOBID] 释放资源，防止机时费空跑。
  • 存储共享：登录节点和计算节点共享 /home 目录，代码只需在一处修改，两处同步生效。
  • 架构一致性：始终确保在 aarch64 环境下编译和安装包，避免混入 x86_64 的二进制文件。
